# 数据工程技能

处理、存储和管理大规模数据系统和管道所需的综合技能。

## 🌐 语言
- [English](data.md) | [中文](data_zh.md)

## 介绍

数据工程包含设计、构建和维护基础设施所需的技能，这些基础设施使数据科学家、分析师和业务利益相关者能够从数据中获得洞察力。数据工程师创建数据管道、数据仓库和处理系统，将原始数据转化为有价值的企业智能。这一领域需要对分布式系统、数据库技术和可扩展数据处理技术的深入了解。

## 核心能力

### 编程语言与生态系统
**核心技能**: 精通用于数据处理和操作的常用语言

- **Python**: Pandas、NumPy、SciPy、scikit-learn、Apache Airflow、Dask
- **SQL**: 复杂查询、存储过程、性能优化、窗口函数
- **Scala**: Apache Spark应用程序、函数式编程概念
- **R**: 统计分析、数据可视化、统计建模
- **Java**: Hadoop生态系统、MapReduce作业、数据处理框架
- **Bash/Shell**: 自动化脚本、数据管道编排
- **JavaScript**: Node.js中的数据处理、ETL工具定制

### 大数据技术
**关键技能**: 理解和使用大数据处理框架

- **Apache Spark**: RDD、DataFrame、Spark SQL、MLlib、流处理
- **Apache Hadoop**: HDFS、MapReduce、YARN、生态系统工具
- **Apache Kafka**: 流处理、实时数据摄取、事件流
- **Apache Flink**: 流和批处理、状态管理
- **Apache Hive**: 数据仓库基础设施、类SQL查询
- **Apache Pig**: 数据分析平台、脚本语言
- **Apache HBase**: NoSQL数据库、实时读写访问

### 数据存储与数据仓库
**专业技能**: 管理各种类型的数据存储解决方案

- **关系数据库**: PostgreSQL、MySQL、性能调优、索引
- **NoSQL数据库**: MongoDB、Cassandra、Couchbase、DynamoDB
- **数据湖**: Amazon S3、Azure数据湖、Google Cloud Storage
- **数据仓库**: Snowflake、Amazon Redshift、Google BigQuery、Azure Synapse
- **列式数据库**: Apache Parquet、Apache Kudu、ClickHouse
- **时间序列数据库**: InfluxDB、TimescaleDB、Druid
- **图数据库**: Neo4j、Amazon Neptune、ArangoDB

### ETL/ELT流程与工具
**高级技能**: 构建和管理数据提取、转换和加载流程

- **Apache Airflow**: 工作流调度、DAG设计、监控
- **Talend**: 数据集成平台、可视化作业设计器
- **Informatica**: 企业数据集成、主数据管理
- **Apache NiFi**: 数据流自动化、可视化数据路由
- **Stitch/Fivetran**: 基于云的ELT工具、自动化数据管道
- **自定义ETL脚本**: 用于特定需求的Python/R/Java实现
- **变更数据捕获(CDC)**: 实时数据同步

### 云数据平台
**核心技能**: 利用云原生数据处理和存储解决方案

- **AWS数据服务**: EMR、Glue、Redshift、S3、Lambda、Kinesis
- **Azure数据服务**: Data Factory、Synapse Analytics、HDInsight、数据湖
- **Google Cloud数据**: BigQuery、Dataflow、Dataproc、Firestore
- **云编排**: 在云平台上管理数据工作流
- **无服务器数据处理**: Lambda、Cloud Functions、无服务器架构
- **成本优化**: 有效管理云数据处理成本
- **安全与合规**: 在云环境中实施数据治理

### 数据建模与架构
**专家技能**: 设计高效和可扩展的数据模型

- **维度建模**: 星型模式、雪花模式、事实和维度表
- **数据仓库建模**: 灵活、可追溯的数据建模技术
- **数据集市设计**: 特定主题区域的数据集市
- **规范化与反规范化**: 存储和查询性能之间的权衡
- **分区策略**: 水平和垂直分区
- **索引技术**: 通过索引优化查询性能
- **数据血缘**: 跟踪数据移动和转换

## 新兴技术

### 数据网格架构
- **领域驱动设计**: 去中心化数据所有权和治理
- **数据产品**: 将数据视为具有定义SLA的产品
- **自助数据平台**: 使数据消费者能够独立访问数据
- **联邦计算治理**: 分布式治理模型

### 数据虚拟化
- **虚拟数据层**: 创建跨多个数据源的统一视图
- **实时集成**: 实时组合来自不同来源的数据
- **抽象层**: 隐藏底层数据源的复杂性

### 高级分析平台
- **增强分析**: AI驱动的数据准备和洞察生成
- **嵌入式分析**: 将分析直接集成到业务应用程序中
- **自然语言处理**: 使用自然语言查询数据

## 专业技能

### 系统架构与设计
- **可扩展性规划**: 设计能够处理日益增长的数据量的系统
- **可靠性工程**: 确保数据系统具有容错能力
- **性能优化**: 优化数据处理和查询性能
- **灾难恢复**: 为数据系统故障和恢复制定计划
- **容量规划**: 估算数据处理的资源需求

### 数据质量与治理
- **数据质量框架**: 实施确保数据准确性的流程
- **主数据管理**: 维护一致的参考数据
- **数据血缘跟踪**: 了解数据流和转换历史
- **合规与隐私**: 实施GDPR、CCPA和其他数据隐私法规
- **数据目录**: 维护元数据和数据字典
- **数据安全**: 实施访问控制和加密

### 问题解决与分析
- **性能分析**: 识别数据处理管道中的瓶颈
- **根本原因分析**: 调查数据质量和系统问题
- **系统监控**: 实施和解释监控解决方案
- **容量规划**: 预测资源需求和扩展要求
- **成本分析**: 分析和优化数据基础设施成本
- **故障排除**: 解决复杂的数据系统问题

### 沟通与协作
- **利益相关者沟通**: 向业务用户解释数据架构和限制
- **跨团队协作**: 与数据科学家、分析师和业务团队合作
- **技术文档**: 记录数据流程和架构
- **知识共享**: 向非技术用户教授数据概念
- **项目管理**: 协调复杂的数据工程项目

## 职业路径

### 初级数据工程师
- 实施基本ETL流程和数据转换
- 使用已建立的数据管道和数据库工作
- 专注于数据质量和正确性
- 学习数据建模和SQL优化
- 与数据分析师和科学家合作

### 数据工程师
- 从头设计和构建数据管道
- 优化现有的数据流程和工作流
- 实施数据质量检查和监控
- 使用大数据技术和云平台
- 与跨职能团队合作

### 高级数据工程师
- 架构复杂的数据系统和数据湖解决方案
- 对数据工具和技术做出技术决策
- 指导初级数据工程师
- 领导数据工程项目和倡议
- 推动数据治理和质量标准

### 首席数据工程师/架构师
- 定义组织范围的数据架构策略
- 评估和选择数据技术与平台
- 推动数据工程实践的创新
- 建立数据治理和安全标准
- 为数据工程团队设定技术方向
