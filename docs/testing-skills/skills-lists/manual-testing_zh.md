# 手动测试技能

专门从事在没有自动化的情况下测试软件应用程序的测试员必备技能，依靠人工观察和交互。

## 🌐 语言
- [English](manual-testing.md) | [中文](manual-testing_zh.md)

## 介绍

手动测试仍然是软件测试过程中的关键组成部分，重点是从人类角度评估软件功能、可用性和用户体验。手动测试员带来自动化无法复制的人类视角，通过真实用户的视角检查应用程序。这一角色需要敏锐的细节注意力、分析性思维以及对软件行为和用户期望的深入理解。

## 核心能力

### 测试用例设计与执行
**关键技能**: 基于需求和规范创建和执行详细的测试用例

- **测试用例创建**: 编写包含明确步骤和预期结果的综合测试用例
- **需求分析**: 理解功能性和非功能性需求
- **测试场景识别**: 识别所有可能的测试场景和边界情况
- **可追溯性矩阵**: 维护需求和测试用例之间的可追溯性
- **测试数据准备**: 创建现实和全面的测试数据集
- **探索性测试**: 临时测试以发现计划测试未涵盖的缺陷
- **边界值分析**: 测试边界条件下的值

### 软件测试基础
**核心技能**: 软件测试的核心原则和实践

- **测试概念**: 了解不同类型的测试(功能、非功能等)
- **测试级别**: 单元、集成、系统和验收测试
- **测试类型**: 黑盒、白盒、灰盒测试方法
- **SDLC集成**: 了解测试如何适应软件开发生命周期
- **质量保证与质量控制**: 区分QA和QC活动
- **缺陷生命周期**: 了解缺陷报告和解决的各个阶段
- **基于风险的测试**: 基于风险评估确定测试优先级

### 可用性和用户体验测试
**专业技能**: 从用户角度评估软件

- **可用性启发式**: 应用Jakob Nielsen的可用性原则
- **无障碍测试**: 确保应用程序符合无障碍标准(WCAG)
- **用户旅程映射**: 了解和测试完整的用户工作流程
- **界面评估**: 评估用户界面的直观性和效率
- **跨平台一致性**: 测试跨平台的一致性体验
- **本地化测试**: 确保应用程序在不同地区正常工作
- **用户验收标准**: 根据用户期望验证软件

### 兼容性和配置测试
**高级技能**: 在不同环境和配置下测试软件

- **浏览器兼容性**: 在不同浏览器和版本间测试应用程序
- **操作系统测试**: 确保与不同操作系统版本的兼容性
- **设备测试**: 在各种设备(桌面、平板、手机)上测试
- **屏幕分辨率测试**: 验证不同分辨率下的UI行为
- **网络条件测试**: 在不同网络速度和条件下测试
- **配置测试**: 验证应用程序在不同配置下的行为
- **向后兼容性**: 确保与较老系统版本的兼容性

### 文档与报告
**专家技能**: 维护全面的测试文档和缺陷报告

- **测试计划创建**: 制定概述范围和方法的综合测试计划
- **测试执行报告**: 记录测试执行结果和覆盖率
- **缺陷报告**: 创建详细、可重现的缺陷报告，包含明确步骤
- **可追溯性文档**: 维护需求、测试用例和缺陷之间的链接
- **测试证据捕获**: 捕获屏幕截图、日志和其他支持证据
- **状态报告**: 提供关于测试进度和质量的定期更新
- **经验教训文档**: 捕获用于未来测试改进的见解

## 专业化轨道

### 用户体验(UX)测试员
- **人为因素**: 了解用户交互的心理学方面
- **认知走查**: 通过认知过程评估可用性
- **A/B测试支持**: 协助用户体检比较测试
- **用户研究**: 进行访谈和调查以了解用户需求
- **人物测试**: 根据定义的用户人物测试应用程序
- **无障碍专家**: 在为残疾用户测试方面的专业知识

### 兼容性专家
- **跨平台测试**: 在多个平台和设备上测试的专业知识
- **浏览器专业知识**: 深入了解浏览器特定的行为和特点
- **移动测试**: 移动平台差异的专业知识
- **遗留系统测试**: 在较老系统和浏览器上测试应用程序
- **国际化测试**: 确保应用程序的全球适用性
- **环境性能**: 在各种配置下测试性能

### 质量分析师
- **流程改进**: 识别并实施测试流程改进
- **指标与KPI**: 开发和跟踪质量指标
- **风险评估**: 识别和评估质量风险
- **质量规划**: 制定质量标准和流程
- **审计与合规**: 确保遵守质量标准
- **培训与指导**: 教育团队质量实践

## 专业技能

### 分析与批判性思维
- **问题识别**: 快速识别潜在问题和异常
- **根本原因分析**: 识别缺陷的根本原因
- **模式识别**: 发现重复出现的问题和趋势
- **逻辑推理**: 遵循逻辑序列来隔离问题
- **细节注意**: 发现细微的不一致和错误
- **批判评估**: 评估已识别问题的重要性

### 沟通与协作
- **利益相关者沟通**: 有效传达测试结果和质量状态
- **缺陷倡导**: 清楚阐述缺陷的影响和重要性
- **跨职能协作**: 与开发和产品团队有效合作
- **反馈提供**: 以外交方式提供建设性反馈
- **文档技能**: 创建清晰、全面的文档
- **演示技能**: 向各种受众呈现测试结果

### 时间管理与组织
- **优先级排序**: 基于风险和影响有效优先排序测试任务
- **测试规划**: 组织测试活动以满足项目时间表
- **效率优化**: 在时间限制内最大化测试效果
- **进度跟踪**: 监控和报告测试进度
- **资源配置**: 优化测试资源和时间的使用
- **截止日期管理**: 在不影响质量的情况下满足测试截止日期

- **文字处理器**: Microsoft Word、Google Docs用于详细文档
- **电子表格应用程序**: Excel、Google Sheets用于测试执行跟踪
- **图表工具**: Lucidchart、Draw.io用于流程和工作流程图
- **版本控制**: Git、SVN用于管理测试文档版本

- **通信平台**: Slack、Microsoft Teams用于团队协调

## 职业路径

### 初级手动测试员
- 执行分配的测试用例并报告基本缺陷
- 学习测试基础和组织流程
- 专注于理解需求和规范
- 在缺陷报告和文档方面建立熟练程度
- 与高级测试员和开发人员协作

### 手动测试员
- 设计和执行综合测试场景
- 识别和报告带有详细步骤的缺陷
- 执行可用性和兼容性测试
- 维护测试文档和可追溯性
- 与跨职能团队协作

### 高级手动测试员
- 在复杂项目上领导手动测试工作
- 指导初级测试员并分享测试知识
- 设计综合测试策略和计划
- 执行基于风险的测试和优先级排序
- 推动流程改进和质量指标

### 手动测试主管/经理
- 监督多个项目的手动测试活动
- 定义手动测试标准和最佳实践
- 管理测试团队和资源
- 推动质量倡议和改进
- 建立质量指标和报告标准

## 结论

手动测试在软件质量保证中继续发挥重要作用，提供自动化测试无法复制的人类洞察。在这一领域取得成功需要对细节的关注、分析性思维以及通过用户视角看待软件的能力。本指南中概述的技能为各级别的手动测试员提供了全面的路线图，从学习基本测试概念的初学者到进行复杂可用性评估的专家。
