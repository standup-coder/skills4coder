# 中级开发者技能清单

这份清单涵盖了有1-3年经验的开发者应该掌握的技能，他们可以构建中等复杂度的应用程序。

## 🌐 语言
- [English](skills_en.md) | [中文](skills_zh.md) | [综合版](skills.md)

## 设计模式

- [ ] 单例模式 - 确保一个类只有一个实例并提供全局访问
- [ ] 观察者模式 - 在对象间定义一对多依赖关系，当一个对象改变状态时，所有依赖项都会收到通知
- [ ] 工厂模式 - 定义创建对象的接口，但让子类决定将创建哪种对象
- [ ] 装饰器模式 - 动态地给对象附加额外职责
- [ ] 适配器模式 - 将一个类的接口转换为客户期望的另一个接口

## 面向对象编程

- [ ] 封装 - 将数据和操作数据的方法绑定在单一单元内
- [ ] 继承 - 基于现有类创建新类以促进代码复用
- [ ] 多态 - 允许不同类型对象被视为同一类型的实例
- [ ] 抽象 - 隐藏复杂实现细节，仅暴露基本特征
- [ ] SOLID 原则 - 理解面向对象设计的五个原则

## 软件架构

- [ ] MVC 架构 - 将应用程序分离为三个相互连接的组件
- [ ] 分层架构 - 将代码组织成具有特定责任的不同层
- [ ] 微服务概念 - 理解将应用程序分解为较小服务的原则
- [ ] RESTful API 设计 - 创建符合 REST 约束的 API
- [ ] 事件驱动架构 - 设计响应事件或状态变化的系统

## 性能优化

- [ ] 内存管理 - 理解内存分配和管理方式
- [ ] 缓存策略 - 实施各种缓存机制以提高性能
- [ ] 数据库查询优化 - 编写高效查询并使用适当索引
- [ ] 代码拆分 - 将大型代码库拆分为更小的部分以加快加载速度
- [ ] 异步编程 - 管理不阻塞主线程的操作

## 测试策略

- [ ] 集成测试 - 测试不同模块或服务之间的交互
- [ ] 端到端测试 - 测试从头到尾的完整用户工作流
- [ ] 使用模拟和桩 - 为测试目的创建模拟对象
- [ ] 测试覆盖率 - 衡量和改进测试执行的代码量
- [ ] 自动化测试管道 - 在开发工作流中实施持续测试

## Web 开发进阶

- [ ] 状态管理 - 在复杂应用程序中管理数据流和应用状态
- [ ] 安全最佳实践 - 实施保护应用程序免受威胁的措施
- [ ] 响应式设计 - 创建在各种屏幕尺寸上都能良好工作的布局
- [ ] 前端性能优化 - 改进加载时间和运行时性能
- [ ] 跨浏览器兼容性 - 确保在不同浏览器中的一致行为

## 数据库高级概念

- [ ] 索引策略 - 通过适当索引优化数据库性能
- [ ] 事务管理 - 在复杂操作期间确保数据完整性
- [ ] 数据库规范化 - 组织数据以减少冗余并提高完整性
- [ ] NoSQL 数据库 - 理解非关系型数据库概念和用例
- [ ] 数据库备份与恢复 - 实施数据保护策略

## DevOps 基础

- [ ] CI/CD 概念 - 理解持续集成和持续部署工作流
- [ ] 容器化 - 将应用程序和依赖项打包到容器中
- [ ] 云服务概念 - 理解云计算模型和服务
- [ ] 监控和日志 - 实施跟踪应用程序健康状况和性能的系统
- [ ] 基础自动化 - 自动化重复的开发和部署任务

2. [Robert Martin 的《代码整洁之道》](https://www.amazon.com/Clean-Code-Handbook-Software-Craftsmanship/dp/0132350882) - 编写可维护代码的最佳实践

4. [Google 技术写作](https://developers.google.com/tech-writing) - 有效技术文档指南
5. [《务实的程序员》](https://pragprog.com/titles/tpp20/the-pragmatic-programmer-20th-anniversary-edition/) - 软件专业人士的永恒建议
