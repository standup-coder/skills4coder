# 高级开发者技能清单

这份清单涵盖了有经验的开发者准备应对复杂挑战所需的技能。

## 🌐 语言
- [English](skills_en.md) | [中文](skills_zh.md) | [综合版](skills.md)

## 系统设计

- [ ] 可扩展架构设计 - 创建能够高效处理增加负载的系统
- [ ] 负载均衡策略 - 在多个服务器间分配网络流量
- [ ] 微服务架构 - 将应用程序设计为松散耦合服务的集合
- [ ] 事件溯源 - 将状态变化存储为事件序列
- [ ] CQRS（命令查询职责分离）- 分离读写操作

## 安全

- [ ] 身份验证和授权 - 实施安全的用户身份识别和访问控制
- [ ] OAuth 和 JWT 实现 - 使用行业标准的身份验证协议
- [ ] 安全漏洞评估 - 识别潜在安全弱点
- [ ] 加密和数据保护 - 保护静态和传输中的敏感数据
- [ ] 安全编码实践 - 编写防止常见安全缺陷的代码

## 性能工程

- [ ] 分布式系统性能 - 优化跨多个系统组件的性能
- [ ] 数据库分片 - 水平分区数据库以提高性能
- [ ] 缓存优化 - 实施高级缓存策略以实现最大效率
- [ ] 内存分析 - 分析内存使用情况以识别性能瓶颈
- [ ] 异步处理 - 管理非阻塞操作以获得更好性能

## 高级架构模式

- [ ] 领域驱动设计 (DDD) - 使软件架构与业务领域保持一致
- [ ] 六边形架构 - 创建具有清晰边界且可测试的灵活应用程序
- [ ] 整洁架构 - 将业务逻辑与外部关注点分离
- [ ] 软件架构的 C4 模型 - 使用分层图表可视化软件架构
- [ ] API 网关实现 - 通过中央点管理微服务通信

## 云和基础设施

- [ ] 容器编排 (Kubernetes) - 大规模管理容器化应用程序
- [ ] 基础设施即代码 (Terraform, CloudFormation) - 通过代码管理基础设施
- [ ] 无服务器架构 - 构建无需管理服务器的应用程序
- [ ] 多云部署策略 - 跨多个云提供商部署应用程序
- [ ] 灾难恢复规划 - 确保故障时的业务连续性

## 数据工程

- [ ] 大数据技术 (Spark, Kafka) - 处理和分析大型数据集
- [ ] 数据管道构建 - 构建可靠的数据处理工作流
- [ ] 实时分析 - 实时处理和分析数据
- [ ] 数据仓库解决方案 - 为分析工作负载设计系统
- [ ] ETL 流程设计 - 高效地提取、转换和加载数据

## 领导力和指导

- [ ] 技术领导力 - 指导技术决策和策略
- [ ] 代码审查最佳实践 - 对代码质量提供建设性反馈
- [ ] 指导初级开发者 - 支持和培养经验较少的团队成员
- [ ] 技术文档 - 创建清晰、全面的文档
- [ ] 知识转移 - 在团队内有效分享知识

## DevOps & SRE

- [ ] 站点可靠性工程实践 - 将软件工程应用于运维
- [ ] 混沌工程 - 主动测试系统弹性
- [ ] 可观测性实现 - 监控、日志记录和跟踪系统行为
- [ ] 自动化部署管道 - 简化发布流程
- [ ] 事件响应 - 处理系统故障和安全事件

2. [AWS 良构框架](https://aws.amazon.com/architecture/well-architected/) - 云架构最佳实践
3. [站点可靠性工程之书](https://sre.google/sre-book/) - Google 的 SRE 实践
4. [设计数据密集型应用](https://dataintensive.net/) - 数据系统架构概念
5. [企业集成模式](https://www.enterpriseintegrationpatterns.com/) - 集成设计模式
